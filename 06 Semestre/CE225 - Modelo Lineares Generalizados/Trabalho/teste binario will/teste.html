1. Base de Dados

Os dados foram retirados do pacote DAAG, sendo dados dos EUA, entre 1997-2002, de acidentes de carro relatados pela polcia nos quais h um evento prejudicial (pessoas ou propriedade) e do qual pelo menos um veculo foi rebocado. Os dados so restritos aos ocupantes do banco da frente, incluem apenas um subconjunto das variveis registradas e so restritos de outras maneiras tambm.

1.1 Descrio dos dados

A base original possui uma base de dados com 26.217 observaes nas 15 variveis a seguir:

1 - dvcat: velocidades estimadas do impacto do acidente: 1-9km/h, 10-24, 25-39, 40-54, 55+
2 - weight: Pesos de observao
3 - dead: Classificao se sobreviveu ao acidente: 1 = sobreviveu ou 0 = morreu
4 - airbag: Se o carro possui airbag: com ou sem airbag
5 - seatbelt: Uso do cinto de segurana: com ou sem cinto
6 - frontal: Impacto do acidente: 0 = no frontal, 1 = impacto frontal
7 - sex: Sexo: 0 = Feminino ou 1 = Masculino
8 - ageOFocc: Idade dos ocupantes do veculo
9 - yearacc: Ano do acidente (1997-2002)
10 - yearVeh: Ano do veculo (1953-2003)
11 - abcat: Se Airbags foram acionados: deploy, nodeploy, unavail
12 - occRole: Posio do airbag acionado: driver, pass
13 - deploy: Airbag acionados: 0: Se no possuia airbag ou no foi acionado, 1: Um ou mais airbags foram acionados
14 - injSeverity: Gravidade do acidente: 0:none, 1 = Possvel Leso, 2:no incapacity, 3:incapacity, 4:killed; 5:unknown, 6:prior death
15 - caseid: ID do caso.

O objetivo da anlise foi modelar a probalidade de sobrevivencia dos acidentes de carro da base nassCD sob a influncia do airbag e outros elementos.

O escopo da anlise tem como respota a varivel dead, que informa se o ocupante do veculo sobreviveu ou no aps o acidente. As demais covariveis sero explicativas, no entando, foi constatado, verificando os dados da base, que algumas delas so irrelevantes ou esto mal explicadas (weight e caseid). Foram retiradas tambm as variveis airbag, ageOFocc, yearVeh, abcat e injSeverity, pois no convergiram ao rodar os modelos.

Outro detalhe, devido ao nmero elevado de registros (mais de 26000 linhas), pegamos uma amostra de aproximadamente 10% do total de registros.

3. Ajuste do Modelo de Regresso

3.1 Ligao Logito

Vamos ajustar um Modelo Linear Generalizado Binomial com funo de ligao Logito. A expresso do modelo dada por:

\(ln (\frac{\pi_i}{1-\pi_i}) = \beta_0 + \beta_1 Veloc_i + \beta_2 Cinto_i + \beta_3 Frontal_i + \beta_4 Sexo_i + \beta_5 Idade_i + \beta_6 Ocupantes_i + \beta_7 AbFunc_i\)

No R, o modelo declarado da seguinte forma:

ajuste1 <- glm(sobrev ~ .,family=binomial(link='logit'),data = dados)

3.2 Ligao Probito

Vamos ajustar um Modelo Linear Generalizado Binomial com funo de ligao Probito. A expresso do modelo dada por:

\(\phi^{-1} (\pi_i) = \beta_0 + \beta_1 Veloc_i + \beta_2 Cinto_i + \beta_3 Frontal_i + \beta_4 Sexo_i + \beta_5 Idade_i + \beta_6 Ocupantes_i + \beta_7 AbFunc_i\)

No R, o modelo declarado da seguinte forma:

ajuste2 <- glm(sobrev ~ .,family=binomial(link = 'probit'),data = dados)

3.3 Ligao Complemento log-log

Vamos ajustar um Modelo Linear Generalizado Binomial com funo de ligao Complemento Log Log. A expresso do modelo dada por:

\(ln[-ln(1-\pi_i)] = \beta_0 + \beta_1 Veloc_i + \beta_2 Cinto_i + \beta_3 Frontal_i + \beta_4 Sexo_i + \beta_5 Idade_i + \beta_6 Ocupantes_i + \beta_7 AbFunc_i\)

No R, o modelo declarado da seguinte forma:

ajuste3 <- glm(sobrev ~ .,family=binomial(link='cloglog'),data = dados)

3.4 Ligao Cauchy

Vamos ajustar um Modelo Linear Generalizado Binomial com funo de ligao Cauchy. A expresso do modelo dada por:

\(tan[\pi_i(\mu_i- 0,5)] = \beta_0 + \beta_1 Veloc_i + \beta_2 Cinto_i + \beta_3 Frontal_i + \beta_4 Sexo_i + \beta_5 Idade_i + \beta_6 Ocupantes_i + \beta_7 AbFunc_i\)

No R, o modelo declarado da seguinte forma:

ajuste4 <- glm(sobrev ~ .,family=binomial(link='cauchit'),data = dados)
